# ACS 반응이 작동하지 않는 이유 분석 결과

## 사용자 질문

**"왜 ACS 반응으로 안돌아가나요? 어차피 ACS를 통해 AMP가 만들어져도 ATP + AMP로 ADP + ADP 만들 수 있잖아요?"**

→ 사용자의 지적은 **이론적으로 완전히 맞습니다!**

---

## 확인된 사실

### 1. ADK1 (Adenylate kinase) 반응 존재 확인

**신규 모델**:
- ✅ ADK1 반응 존재: `AMP + ATP <=> 2ADP`
- ✅ bounds: [-1000.0, 1000.0] (정상)

**레퍼런스 모델**:
- ✅ ADK1 반응 존재
- ✅ ATPM=0일 때 플럭스: 0.2096 (작동 중)

### 2. 레퍼런스 모델의 실제 사용 반응

레퍼런스 모델의 FBA 결과 (`fba_flux_gradient_acid.csv`):

| 반응 | ATPM=0 플럭스 | 사용 여부 |
|------|--------------|----------|
| **ACS** | 0.0 | ❌ 사용 안 함 |
| **ACS_ADP** | 0.943448 | ✅ **실제 사용** |
| **ADK1** | 0.2096 | ✅ 사용 중 |

---

## 핵심 발견

**레퍼런스 모델도 ACS를 사용하지 않고 ACS_ADP를 사용합니다!**

### 왜 ACS를 사용하지 않을까?

1. **에너지 효율성 문제**:
   - **ACS**: `ATP → AMP + PPi` (ATP 1개 → AMP 생성)
   - **ADK1 재생성**: `AMP + ATP → 2ADP` (AMP를 ADP로 전환하려면 추가 ATP 필요)
   - 결과: ACS는 **에너지 효율이 낮음**

2. **ACS_ADP의 장점**:
   - **ACS_ADP**: `ATP → ADP + Pi` (ATP 1개 → ADP 직접 생성)
   - **에너지 효율**: ACS_ADP가 더 효율적
   - 결과: 최적화 과정에서 ACS_ADP를 선택

3. **최적화 결과**:
   - FBA는 최적 성장률을 목표로 함
   - 에너지 효율이 낮은 ACS 대신 효율적인 ACS_ADP를 선택
   - 따라서 ACS는 플럭스가 0

---

## 신규 모델의 문제

### 현재 상황

| 항목 | 신규 모델 | 레퍼런스 모델 |
|------|----------|--------------|
| **ACS** | ✅ 있음 (플럭스: 0) | ✅ 있음 (플럭스: 0) |
| **ACS_ADP** | ❌ **없음** | ✅ 있음 (플럭스: 0.943448) |
| **ADK1** | ✅ 있음 | ✅ 있음 |

### 문제점

1. **ACS_ADP 반응이 없음**
   - 레퍼런스 모델은 ACS_ADP를 사용
   - 신규 모델에는 ACS_ADP가 없음
   - 따라서 Acetate → Acetyl-CoA 전환이 불가

2. **ACS는 이론적으로 작동 가능하지만...**
   - 사용자 지적대로 ADK1이 있으면 ACS로도 작동 가능
   - 하지만 에너지 효율이 낮아서 FBA 최적화에서 선택되지 않음
   - 레퍼런스 모델도 ACS를 사용하지 않음

---

## 결론

### 사용자 지적이 맞습니다!

✅ **이론적으로**: ACS (ATP → AMP + PPi) + ADK1 (AMP + ATP → 2ADP)로 작동 가능

하지만 **실제로는**:
- 레퍼런스 모델도 ACS를 사용하지 않고 ACS_ADP를 사용
- 에너지 효율이 낮아서 FBA 최적화에서 선택되지 않음
- 신규 모델에도 ACS_ADP가 없어서 Acetate 전환이 불가

### 해결 방안

**ACS_ADP 반응 추가가 필요합니다!**

1. **ACS_ADP 추가**:
   - 반응식: `ac_c + atp_c + coa_c <=> accoa_c + adp_c + pi_c`
   - 레퍼런스 모델에서 실제 사용하는 반응
   - 에너지 효율이 좋아서 FBA 최적화에서 선택됨

2. **이론적 대안**:
   - ACS + ADK1 조합도 작동 가능 (사용자 지적대로)
   - 하지만 에너지 효율이 낮아서 실제로는 사용되지 않음
   - 레퍼런스 모델도 사용하지 않음

---

## 요약

1. ✅ 사용자 지적: ACS + ADK1로 작동 가능 (이론적으로 맞음)
2. ✅ ADK1은 신규 모델에 이미 존재
3. ✅ 레퍼런스 모델도 ACS를 사용하지 않음 (플럭스: 0)
4. ✅ 레퍼런스 모델은 ACS_ADP를 사용 (플럭스: 0.943448)
5. ❌ 신규 모델에 ACS_ADP가 없음
6. ✅ 해결: ACS_ADP 반응 추가 필요

**결론**: 사용자의 이론적 지적은 맞지만, 실제로는 ACS_ADP가 더 효율적이어서 사용됩니다. 신규 모델에 ACS_ADP를 추가해야 합니다!
