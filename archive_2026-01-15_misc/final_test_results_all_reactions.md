# 레퍼런스 모델 반응 추가 후 FBA 테스트 결과

## 테스트 개요

레퍼런스 모델(`model_YE0p5.xml`)에 있지만 신규 모델(`BaseModel.xml`)에 없는 모든 반응 114개를 우선순위별로 추가하며 FBA를 테스트한 결과입니다.

- **성장률**: 0.000000

### Step 2: HIGH 우선순위 반응 추가
- **추가된 반응 수**: 0개 (HIGH는 이미 실제 사용된 반응에 포함됨)
- **FBA 상태**: OPTIMAL_BUT_ZERO
- **성장률**: 0.000000

### Step 3: MEDIUM 우선순위 반응 추가 (90개)

## 최종 결과

### 전체 추가 결과
- **추가된 총 반응 수**: 114개
- **최종 FBA 상태**: OPTIMAL_BUT_ZERO
- **최종 성장률**: 0.000000

### 결론

**레퍼런스 모델의 모든 누락 반응(114개)을 추가해도 FBA가 실패합니다.**

## 분석

### 주요 발견 사항

1. **모든 반응 추가 완료**
   - 114개 반응 모두 성공적으로 추가됨
   - 최적해는 찾을 수 있지만 성장률이 0

2. **구조적 차이**
   - 레퍼런스 모델의 반응을 추가하는 것만으로는 부족
   - 신규 모델의 구조나 대사물질 연결에 문제가 있을 수 있음
   - 또는 신규 모델에만 있는 특정 반응/경로가 필요할 수 있음

3. **가능한 원인**
   - 대사물질 연결 문제
   - 신규 모델에 고유한 경로나 반응 필요
   - 모델 구조의 근본적인 차이
   - Biomass 구성 요소 생산 경로의 차이

## 권장 사항

### 다음 단계

1. **신규 모델 고유 분석**
   - 신규 모델에만 있는 반응/경로 확인
   - 신규 모델의 Biomass 구성 요소 확인
   - 신규 모델의 대사 네트워크 구조 분석

2. **상세 진단**
   - Blocked reactions 상세 분석
   - Biomass 구성 요소별 생산 경로 확인
   - 대사물질 연결 확인
   - 핵심 경로 (Acetate → Biomass) 추적

3. **대안 접근**
   - Gap-filling 도구 사용
   - 신규 모델의 실제 유전자 주석 기반 반응 추가
   - 레퍼런스 모델의 반응 추가가 아닌, 신규 모델 구조에 맞는 반응 찾기

## 결론

레퍼런스 모델의 반응을 추가하는 것만으로는 신규 모델에서 FBA가 성공하지 않습니다.

신규 분리 미생물의 GSM은 레퍼런스 균주와 구조적으로 차이가 있으며, 단순히 반응을 추가하는 것보다는 신규 모델의 고유한 특성을 고려한 접근이 필요합니다.
