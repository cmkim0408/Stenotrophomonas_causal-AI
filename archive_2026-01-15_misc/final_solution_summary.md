## 핵심 발견

### 1. CoA 합성 경로 문제

**발견**:
- 신규 모델: 주요 CoA 합성 반응 0개
- 레퍼런스 모델: 주요 CoA 합성 반응 0개
- **두 모델 모두 CoA 합성 경로가 없음!**

**의미**:
- CoA는 초기화되어 있거나 다른 방식으로 공급되어야 함
- 레퍼런스 모델도 CoA 합성 경로가 없지만 작동함
- **다른 초기화 메커니즘이 있을 수 있음**

### 2. Pantothenate 제공해도 CoA 생성 불가

**결과**:
- Pantothenate 부트스트랩 제공
- Transport 반응 추가
- 하지만 CoA 생성 불가

**원인 추정**:
- CoA 합성 경로 자체가 모델에 없음
- 또는 CoA 합성에 필요한 다른 전구체가 없음

### 3. 레퍼런스 모델의 차이점

**발견**:
- 레퍼런스 모델에만 있는 Transport:
  - `T_coa_e_to_coa_c`: CoA transport
  - `T_pnto__R_e_to_c`: Pantothenate transport
  - `EX_pnto__R_e`: Pantothenate exchange

**추가 시도**:
- 메타볼라이트 추가: coa_e, pnto__R_e, pnto__Rc
- Transport 반응 추가
- 하지만 여전히 작동 안 함

## 문제 분석

### 왜 레퍼런스 모델은 작동하는가?

**가능한 이유**:
1. **CoA가 이미 초기화되어 있음**
   - 레퍼런스 모델의 초기 상태에 CoA가 포함되어 있을 수 있음
   - 또는 다른 경로로 CoA가 공급됨

2. **다른 CoA 공급 경로**
   - CoA 합성 경로가 아닌 다른 방식으로 CoA 공급
   - 예: 다른 메타볼라이트에서 CoA 생성

3. **모델 구조 차이**
   - 레퍼런스 모델의 구조가 다를 수 있음
   - 제약 조건이 다를 수 있음

### 신규 모델의 문제

**확인된 문제**:
1. CoA 생성 불가
2. CoA 합성 경로 없음
3. Transport 추가해도 작동 안 함

**추정 원인**:
- CoA 초기화 메커니즘이 없음
- 또는 CoA 합성에 필요한 모든 반응이 누락됨

## 해결 방안

### 방안 1: CoA 초기화 확인

**확인 필요**:
- 레퍼런스 모델에서 CoA가 어떻게 초기화되는지
- 초기 상태에 CoA가 포함되어 있는지
- 다른 메타볼라이트에서 CoA를 생성하는지

### 방안 2: CoA 합성 경로 추가

**필요 작업**:
- CoA 합성 경로의 모든 반응 확인
- 누락된 반응 추가
- 전구체 공급 경로 확인

### 방안 3: 레퍼런스 모델의 실제 초기화 방식 확인

**확인 필요**:
- 레퍼런스 모델의 FBA 설정 확인
- 초기 메타볼라이트 확인
- 부트스트랩 설정 확인
